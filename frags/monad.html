A pesar de ser un concepto principalmente usado en el paradigma funcional, las mónadas resultan muy útiles para reducir la cantidad de código que tendremos que escribir para una serie de modificaciones en un mismo objeto.
Al operar dentro de un objeto, podemos utilizar la palabra this para referenciar el objeto mismo más inmediato:
miObjeto = {
private:
valUno = 4
public:
cambiarValUno = function (delta) {
valUno += delta;
return this;
}
}
miObjeto.cambiarValUno(2); // miObjeto.valUno va a ser igual a 6
copiaMiObjeto = miObjeto.cambiarValUno(4); // copiaMiObjeto.valUno & miObjeto.valUno van a ser iguales a 10
copiaMiObjeto.cambiarValUno(9); // copiaMiObjeto.valUno va a ser igual a 19, pero miObjeto.valUno se mantiene en 10
Al producir el mismo objeto, todos los operadores que se le apliquen a la instrucción, así como las funciones a las que se les pase, van a tratar con el objeto entero, no solo con una propiedad o un valor nulo. Siendo que los operadores punto-propiedad y puntero-propiedad (obj.propOmet y obj->propOmet) son, justamente, operadores, podemos concatenarlos tal que:
miObjeto.cambiarValUno(3).cambiarValUno(-2).cambiarValUno(7).cambiarValUno(92); // miObjeto.valUno, si lo declaramos como 4, acabará como 104
En el paradigma funcional, las mónadas toman todavía más protagonismo y expansión, siendo que podríamos declarar un flujo de datos completo en tan sólo unas líneas con valores que no necesariamente son objetos; Haskell hace amplio uso de tales.
