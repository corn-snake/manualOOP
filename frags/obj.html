Como ya mencionamos, un mapa es la forma más simple de objeto. Pero, ¿qué es exactamente un objeto?

Pensemos en como describimos un libro: cómo es su portada, cuántas hojas tiene, su autoría, precio, número ISBN, y un largo etcétera.
Cada una de estas se puede considerar como una propiedad, que podemos representar en forma de lista:
libro = {
portada = {
color = azul,
tipo = pasta dura,
bestseller = sí
},
nombre = The Water Dancer,
autoría = Ta-Nehisi Coates,
...
}
Como podemos observar, cada propiedad tiene un nombre y un valor, como una variable — son un par clave-valor.
Lo interesante de los objetos es que también contienen funciones internas, que llamaremos métodos:
libro = {
últimaPáginaLeída = 84;
...
leer = function () {
abrir(),
leer(últimaPáginaLeída + 1),
últimaPáginaLeída += 1;
},
remarcar = function (página, primeraLetra, longitud) {
irA(páginaActual),
escribir(primeraLetra, longitud, { con = marcatextos });
}
}
Cuando creamos un objeto, la máquina va a apartar cierta cantidad de memoria; normalmente vamos a hacer esto de manera estática, con espacios seguidos para sus propiedades, aunque también podemos inicializar una lista vacía que se va a ir llenando con direcciones a propiedades, o bien, dir.s de más listas de direcciones si se añaden o quitan propiedades después de su inicialización.
