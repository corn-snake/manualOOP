Aún así, nos es útil abstraer esto para guiar nuestra lógica y evitar errores.
En general, existen dos tipos de valores esenciales:
  1. Lógicos — Verdadero o Falso, Sí o No, 1 o 0
  2. Números — Dependiendo de la arquitectura (y el registro), cualquier cifra positiva (>=0) que se pueda representar con 8, 16, 32 o 64 bits
Esta lista tiende a expandirse de manera que podamos leer datos un tanto menos mecánicos:
  1. Carácteres — Letras individuales; originalmente solo ∼128, con el desarrollo del estandar UNiCODE ahora son más de un millón, todos cabiendo en un número de tan solo 16 bits.
  2. Números "signados" — Números cuyo primer bit de derecha a izquierda determina su signo; si tu compu es de 32 bits, el n№ máximo de esta categorí que puede procesar en un solo paso es 111,1111,1111,1111,1111,1111 — ±2147483647
  3. Números largos — que usan dos registros para un valor. Pueden ser signados (equipo de 32b -> número de de 63b), y en ciertos lenguajes se puede usar un "número largo-largo," que duplica el número de bits útiles del n. largo regular.
  4. Números de punto flotante — son números signados cuya primera seccióan (8b para la precisión regular, 11b para la especificación "double"/64b) se usa como exponente, y el resto como componente numérico. Se procesan mediante una fórmula bastante compleja, y son más para aproximaciones que para aritmética exacta.
Todos estos juntos son valores "primitivos," así llamados por ser los más simples, cómo las células de un ser.

A partir de estos, existen tipos "complejos":
  • Listas — Colecciones de datos del mismo tipo, cuya longitud es fija.
    • Matrices — Colecciones cuyos elementos son subsecuentes.
    • Listas enlazadas — Colecciónes donde cada dato también contiene un espacio subsecuente, que apunta a la dirección del siguiente elemento.
    • Lista enumerada — Colecciónes cuyos valores de sus datos no se pueden re-escribir una vez inicializados.
  • Vectores — Colecciones cuyo tamaño (pero no tipo) es dinámico; por lo mismo, tienden a ser enlazados
  • Mapas — Colecciones que, en vez de contar elementos de uno en uno, les asigna una clave; en esencia, es una variable que contiene variables
Estas últimas son, en esencia, el tipo más simple de objetos que podemos tener.
