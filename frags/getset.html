Si apartamos memoria para un objeto de manera estática, podremos acceder a una propiedad de la siguiente manera:
variableUno = miObjeto.propiedad; // copiamos valor de la propiedad a una variable independiente
miObjeto.propiedad = 15; // cambiamos manualmente el valor de la propiedad
miObjeto.método(variableUno); // llamamos al método
En cambio, para la manera dinámica:
variableUno = miObjeto->propiedad;
miObjeto->propiedad = 15;
miObjeto->método(variableUno);
Claro, siempre habrá casos en los que tendremos que agregar o quitar propiedades de un objeto, por lo que siempre se almacena un espacio "extra" para almacenar una dirección a alguna segunda lista de propiedades si es que se requiere.
Habrá que mencionar que, como en todo, los diferentes lenguajes varín: fuera de C++, la mayoría sólo usa la sintaxis con el operador ., por lo que esta distinción estático-dinámico es un tanto más técnica que práctica.

Por convención, se evita acceder a las propiedades de un objeto directamente: preferimos añadir métodos que nos apoyen para ello. Aquellos que leen propiedades se les llamaran getters, y aquellos que las cambian son setters, tal que:
miObjeto = {
propUno = 0,
propDos = 15,
propTres = 'veintiseis',
...
getterUno = function () {
return propUno;
}
getterDos = function () {
return propDos;
}
...
setterUno = function (valNuevo) {
if (typeof valNuevo == número) {
propUno = valNuevo;
return propUno;
}
return ErrorDeTipo;
}
...
}
Esto se hace principalmente para tres cosas: para mantener ciertas partes del código constantes y, por ende, predecibles, validación,  encapsulación y, a un nivel más alto, polimorfismo.
